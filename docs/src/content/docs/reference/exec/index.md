Package exec contains the GraphQL lexer, parser, AST, executor, and introspection fast\-path. It is the hot per\-request execution path and is deliberately transport\-agnostic: callers communicate with the executor through \[core.Request\] and \[core.Response\].

<a name="ErrSubscriptionOperation"></a>ErrSubscriptionOperation is returned by [Executor.Execute](<#Executor.Execute>) when the document defines a subscription operation. Subscriptions must be run through [Executor.Subscribe](<#Executor.Subscribe>) so callers receive a streaming channel.

```go
var ErrSubscriptionOperation = errors.New("subscription operations must use the streaming Subscribe entry point")
```

<a name="Executor"></a>
## type Executor

Executor runs GraphQL operations against a built \[schema.Schema\] and notifies the registered plugins at each lifecycle phase. It satisfies \[core.Executor\] and is safe for concurrent use.

```go
type Executor struct {
    Schema  *schema.Schema
    Plugins []plugin.Plugin
}
```

<a name="New"></a>
### func New

```go
func New(schemaValue *schema.Schema, plugins []plugin.Plugin) *Executor
```

New returns an [Executor](<#Executor>) bound to schemaValue and plugins. plugins may be nil; an Executor with no plugins simply skips the lifecycle notifications.

<a name="Executor.Execute"></a>
### func \(\*Executor\) Execute

```go
func (e *Executor) Execute(ctx context.Context, req core.Request) core.Response
```

Execute runs a query or mutation operation and returns the completed response. Subscription operations are rejected with [ErrSubscriptionOperation](<#ErrSubscriptionOperation>); use [Executor.Subscribe](<#Executor.Subscribe>) instead. Errors produced during plugin notifications, parsing, or field resolution are surfaced via the returned response.

<a name="Executor.OperationKind"></a>
### func \(\*Executor\) OperationKind

```go
func (e *Executor) OperationKind(req core.Request) (core.OperationKind, error)
```

OperationKind parses req and reports the kind of the selected operation. It performs no plugin notifications and runs no resolvers, so transports can call it cheaply to decide whether to dispatch a request to Execute or Subscribe.

<a name="Executor.Subscribe"></a>
### func \(\*Executor\) Subscribe

```go
func (e *Executor) Subscribe(ctx context.Context, req core.Request) (<-chan core.Response, error)
```

Subscribe parses a subscription operation, invokes the source\-stream resolver, and returns a channel of GraphQL responses that close when the source stream closes or the supplied context is cancelled.
